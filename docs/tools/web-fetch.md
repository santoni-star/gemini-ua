# Інструмент веб-запитів (`web_fetch`)

Цей документ описує інструмент `web_fetch` для Gemini CLI.

## Опис

Використовуйте `web_fetch`, щоб підсумовувати, порівнювати або отримувати
інформацію з веб-сторінок. Інструмент обробляє вміст за одним або кількома
URL-адресами (до 20), вказаними у вашому запиті.

### Аргументи

- `prompt` (рядок, обов'язково): Повний запит, що включає URL-адреси та
  інструкції щодо їх обробки. Наприклад:
  `"Зроби підсумок статті https://example.com/article та виділи головні тези"`.
  Запит повинен містити принаймні одне посилання, що починається з `http://` або
  `https://`.

## Як працює `web_fetch`

Коли ви вводите запит із посиланням, інструмент запитує підтвердження. Після
згоди він обробляє URL через Gemini API. Якщо API не може отримати доступ до
сайту, інструмент спробує завантажити вміст безпосередньо з вашої локальної
машини (fallback).

Відповідь включатиме посилання на джерела та цитати, де це можливо.

## Приклади

**Підсумок однієї статті:**

```text
web_fetch(prompt="Зроби короткий огляд основних моментів https://example.com/news/latest")
```

**Порівняння двох сторінок:**

```text
web_fetch(prompt="Яка різниця у висновках цих двох робіт: https://arxiv.org/abs/2401.0001 та https://arxiv.org/abs/2401.0002?")
```
