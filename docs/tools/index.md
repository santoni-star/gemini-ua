# Інструменти Gemini CLI

Gemini CLI містить вбудовані інструменти, які модель Gemini використовує для
взаємодії з вашим локальним середовищем, доступу до інформації та виконання дій.
Ці інструменти розширюють можливості CLI, дозволяючи йому виходити за рамки
генерації тексту та допомагати у широкому спектрі завдань.

## Огляд інструментів Gemini CLI

В контексті Gemini CLI інструменти — це специфічні функції або модулі, які
модель Gemini може попросити виконати. Наприклад, якщо ви попросите Gemini
"Зроби підсумок вмісту `document.txt`", модель ідентифікує потребу прочитати цей
файл і зробить запит на виконання інструменту `read_file`.

Компонент ядра (`packages/core`) керує цими інструментами, представляє їхні
визначення (схеми) моделі Gemini, виконує їх за запитом і повертає результати
моделі для подальшої обробки у відповідь користувачеві.

Ці інструменти надають такі можливості:

- **Доступ до локальної інформації:** Інструменти дозволяють Gemini звертатися
  до вашої файлової системи, читати вміст файлів, перелічувати каталоги тощо.
- **Виконання команд:** За допомогою інструментів, таких як `run_shell_command`,
  Gemini може запускати команди терміналу (з відповідними заходами безпеки та
  підтвердженням користувача).
- **Взаємодія з вебом:** Інструменти можуть отримувати вміст за URL-адресами.
- **Виконання дій:** Інструменти можуть змінювати файли, створювати нові або
  виконувати інші дії у вашій системі.
- **Обґрунтування відповідей:** Використовуючи інструменти для отримання
  актуальних або специфічних локальних даних, відповіді Gemini стають точнішими
  та релевантнішими вашому контексту.

## Як працюють інструменти

Процес використання інструментів виглядає так:

1.  Ви вводите запит у Gemini CLI.
2.  CLI надсилає запит до ядра.
3.  Ядро разом із вашим запитом та історією розмови надсилає список доступних
    інструментів та їхніх описів до Gemini API.
4.  Модель Gemini аналізує запит. Якщо вона вирішує, що потрібен інструмент, її
    відповідь міститиме запит на виконання конкретного інструменту з певними
    параметрами.
5.  Ядро отримує цей запит, перевіряє його і (зазвичай після вашого
    підтвердження) виконує інструмент.
6.  Результат роботи інструменту надсилається назад моделі Gemini.
7.  Модель використовує цей результат для формування остаточної відповіді вам.

Ви зазвичай бачитимете повідомлення в CLI про те, який інструмент викликається і
чи успішно він спрацював.

## Безпека та підтвердження

Багато інструментів, особливо ті, що змінюють файли або виконують команди
(`write_file`, `edit`, `run_shell_command`), розроблені з урахуванням безпеки.
Gemini CLI зазвичай:

- **Вимагає підтвердження:** Запитує вас перед виконанням потенційно чутливих
  операцій, показуючи, яку саме дію буде вчинено.
- **Використовує пісочницю:** Усі інструменти підпорядковуються обмеженням
  пісочниці (див. [Пісочниця в Gemini CLI](../cli/sandbox.md)).

## Дізнайтеся більше про інструменти

Вбудовані інструменти Gemini CLI можна розділити на такі категорії:

- **[Інструменти файлової системи](./file-system.md):** Для взаємодії з файлами
  та каталогами.
- **[Інструмент оболонки](./shell.md) (`run_shell_command`):** Для виконання
  команд терміналу.
- **[Інструмент веб-запитів](./web-fetch.md) (`web_fetch`):** Для отримання
  вмісту за URL.
- **[Інструмент веб-пошуку](./web-search.md) (`google_web_search`):** Для пошуку
  в Інтернеті.
- **[Інструмент пам'яті](./memory.md) (`save_memory`):** Для збереження
  інформації між сесіями.
- **[Інструмент завдань (Todo)](./todos.md) (`write_todos`):** Для керування
  підзавданнями.

Також інструменти включають:

- **[Сервери MCP](./mcp-server.md)**: Міст між моделлю та сторонніми сервісами.
- **[Пісочниця](../cli/sandbox.md)**: Ізоляція виконання для зниження ризиків.
