# Інструменти файлової системи Gemini CLI

Gemini CLI надає повний набір інструментів для взаємодії з локальною файловою
системою. Ці інструменти дозволяють моделі Gemini читати, записувати, шукати та
змінювати файли під вашим контролем.

**Примітка:** З міркувань безпеки всі інструменти працюють у межах кореневого
каталогу (зазвичай це папка, з якої ви запустили CLI).

## 1. `list_directory` (ReadFolder)

Виводить імена файлів та підкаталогів у вказаному шляху. Може ігнорувати файли
за шаблонами.

- **Назва інструменту:** `list_directory`
- **Параметри:**
  - `path` (рядок, обов'язково): Абсолютний шлях до каталогу.
  - `ignore` (масив рядків, опціонально): Шаблони glob для виключення
    (наприклад, `["*.log", ".git"]`).
  - `respect_git_ignore` (boolean, опціонально): Чи враховувати `.gitignore`.
    Типово — `true`.
- **Підтвердження:** Ні.

## 2. `read_file` (ReadFile)

Читає та повертає вміст файлу. Підтримує текст, зображення, аудіо та PDF.

- **Назва інструменту:** `read_file`
- **Параметри:**
  - `path` (рядок, обов'язково): Шлях до файлу.
  - `offset` (число, опціонально): Номер рядка, з якого почати читання (для
    тексту).
  - `limit` (число, опціонально): Кількість рядків для читання.
- **Підтвердження:** Ні.

## 3. `write_file` (WriteFile)

Записує вміст у вказаний файл. Якщо файл існує — він буде перезаписаний. Якщо ні
— створений.

- **Назва інструменту:** `write_file`
- **Параметри:**
  - `file_path` (рядок, обов'язково): Шлях до файлу.
  - `content` (рядок, обов'язково): Вміст файлу.
- **Підтвердження:** Так. Показує diff (різницю) та запитує дозвіл.

## 4. `glob` (FindFiles)

Шукає файли за шаблонами glob (наприклад, `src/**/*.ts`, `*.md`). Повертає
шляхи, відсортовані за часом зміни (найновіші спочатку).

- **Назва інструменту:** `glob`
- **Параметри:**
  - `pattern` (рядок, обов'язково): Шаблон для пошуку.
  - `path` (рядок, опціонально): Каталог для пошуку.
- **Підтвердження:** Ні.

## 5. `search_file_content` (SearchText)

Шукає текст за допомогою регулярних виразів усередині файлів. Повертає рядки зі
збігами, шляхи та номери рядків.

- **Назва інструменту:** `search_file_content`
- **Параметри:**
  - `pattern` (рядок, обов'язково): Регулярний вираз для пошуку.
  - `include` (рядок, опціонально): Шаблон glob для фільтрації файлів
    (наприклад, `"*.js"`).
- **Підтвердження:** Ні.

## 6. `replace` (Edit)

Замінює текст у файлі. Створений для точних правок і вимагає контексту (кілька
рядків до та після) навколо тексту, що замінюється.

- **Назва інструменту:** `replace`
- **Параметри:**
  - `file_path` (рядок, обов'язково): Шлях до файлу.
  - `old_string` (рядок, обов'язково): Текст, який треба замінити. Має бути
    унікальним та включати контекст.
  - `new_string` (рядок, обов'язково): Новий текст.
- **Поведінка:** Якщо `old_string` порожній — створює новий файл. Має механізм
  самокорекції, якщо модель помилилася у відступах або точності контексту.
- **Підтвердження:** Так. Показує зміни та запитує схвалення.
