# Інтеграція з IDE

Gemini CLI може інтегруватися з вашою IDE для створення більш зручного та
контекстного досвіду роботи. Це дозволяє CLI краще розуміти ваш робочий простір
і надає такі можливості, як перегляд змін (diff) безпосередньо в редакторі.

Наразі підтримуються [Visual Studio Code](https://code.visualstudio.com/) та
інші редактори, що підтримують розширення VS Code.

## Можливості

- **Контекст робочого простору:** CLI автоматично отримує доступ до інформації
  про ваш проект для надання точніших відповідей. Це включає:
  - **10 останніх файлів**, до яких ви зверталися.
  - Позицію курсору.
  - Виділений текст (до 16 КБ).

- **Перегляд змін (Native diffing):** Коли Gemini пропонує змінити код, ви
  можете побачити різницю безпосередньо у вікні diff вашої IDE. Це дозволяє
  легко перевірити, відредагувати та прийняти або відхилити пропоновані зміни.

- **Команди VS Code:** Ви можете отримати доступ до функцій Gemini CLI через
  палітру команд (`Cmd+Shift+P` або `Ctrl+Shift+P`):
  - `Gemini CLI: Run`: запуск нової сесії у вбудованому терміналі.
  - `Gemini CLI: Accept Diff`: прийняти зміни у вікні diff.
  - `Gemini CLI: Close Diff Editor`: відхилити зміни та закрити вікно.

## Встановлення та налаштування

### 1. Автоматична пропозиція (рекомендовано)

Коли ви запускаєте Gemini CLI у вбудованому терміналі підтримуваного редактора,
він автоматично виявить середовище і запропонує підключитися.

### 2. Встановлення через CLI

Ви можете запустити команду встановлення вручну:

```text
/ide install
```

### 3. Встановлення з маркетплейсу

Знайдіть розширення **Gemini CLI Companion** у
[VS Code Marketplace](https://marketplace.visualstudio.com/items?itemName=google.gemini-cli-vscode-ide-companion)
та встановіть його. Після встановлення введіть `/ide enable` у CLI.

## Використання

- **Увімкнення:** `/ide enable`.
- **Вимкнення:** `/ide disable`.
- **Перевірка статусу:** `/ide status`.

## Усунення несправностей

- **Помилка підключення:** Переконайтеся, що розширення "Gemini CLI Companion"
  встановлене та увімкнене. Відкрийте новий термінал в IDE, щоб оновити змінні
  середовища.
- **Невідповідність каталогів:** Переконайтеся, що ви запустили CLI саме в тій
  папці, яка відкрита у вашій IDE.
