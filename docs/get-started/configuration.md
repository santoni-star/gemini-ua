# Налаштування Gemini CLI

Gemini CLI пропонує кілька способів налаштування своєї поведінки, включаючи
змінні середовища, аргументи командного рядка та файли налаштувань. У цьому
документі описано різні методи конфігурації та доступні параметри.

## Рівні конфігурації

Налаштування застосовуються в такому порядку пріоритетності (менші номери
перекриваються більшими):

1.  **Значення за замовчуванням:** Жорстко запрограмовані параметри всередині
    програми.
2.  **Файл системних значень за замовчуванням:** Загальносистемні налаштування,
    які можуть бути змінені іншими файлами.
3.  **Файл налаштувань користувача:** Глобальні налаштування для поточного
    користувача.
4.  **Файл налаштувань проекту:** Специфічні налаштування для конкретного
    проекту.
5.  **Файл системних налаштувань:** Загальносистемні налаштування, що мають
    пріоритет над іншими файлами (для адміністрування).
6.  **Змінні середовища:** Змінні на рівні системи або сесії, які також можуть
    завантажуватися з файлів `.env`.
7.  **Аргументи командного рядка:** Значення, що передаються безпосередньо при
    запуску CLI.

## Файли налаштувань

Gemini CLI використовує файли JSON для збереження конфігурації. Ці файли можуть
знаходитися у чотирьох місцях:

- **Файл системних значень за замовчуванням:**
  - **Розташування:** `/etc/gemini-cli/system-defaults.json` (Linux),
    `C:\ProgramData\gemini-cli\system-defaults.json` (Windows) або
    `/Library/Application Support/GeminiCli/system-defaults.json` (macOS).
  - **Область дії:** Базовий шар налаштувань для всієї системи. Має найнижчий
    пріоритет.
- **Файл налаштувань користувача:**
  - **Розташування:** `~/.gemini/settings.json` (де `~` — ваш домашній каталог).
  - **Область дії:** Застосовується до всіх сесій Gemini CLI для поточного
    користувача. Перекриває системні значення за замовчуванням.
- **Файл налаштувань проекту:**
  - **Розташування:** `.gemini/settings.json` у кореневому каталозі вашого
    проекту.
  - **Область дії:** Застосовується лише при запуску Gemini CLI з цього
    конкретного проекту. Перекриває налаштування користувача та системні
    значення.
- **Файл системних налаштувань:**
  - **Розташування:** `/etc/gemini-cli/settings.json` (Linux),
    `C:\ProgramData\gemini-cli\settings.json` (Windows) або
    `/Library/Application Support/GeminiCli/settings.json` (macOS).
  - **Область дії:** Застосовується до всіх користувачів системи. Має вищий
    пріоритет над усіма іншими файлами налаштувань.

### Каталог `.gemini` у вашому проекті

Окрім файлу налаштувань, каталог `.gemini` може містити інші файли проекту:

- [Власні профілі пісочниці](#sandboxing) (наприклад,
  `.gemini/sandbox.Dockerfile`).
- Файли контексту (наприклад, `.gemini/system.md`).

## Аргументи командного рядка

Аргументи, передані при запуску, мають пріоритет для конкретної сесії.

- **`--model <назва_моделі>`** (**`-m`**): Вказати модель Gemini.
- **`--prompt <підказка>`** (**`-p`**): Запустити в неінтерактивному режимі з
  вказаною підказкою.
- **`--prompt-interactive <підказка>`** (**`-i`**): Почати інтерактивну сесію з
  початковою підказкою.
- **`--output-format <формат>`**: Формат виводу (`text`, `json`, `stream-json`).
- **`--sandbox`** (**`-s`**): Увімкнути режим пісочниці.
- **`--debug`** (**`-d`**): Увімкнути режим налагодження.
- **`--yolo`**: Режим YOLO (автоматичне схвалення всіх дій).
- **`--resume [session_id]`** (**`-r`**): Відновити попередню сесію.
- **`--include-directories <dir1,dir2,...>`**: Включити додаткові каталоги до
  контексту.

## Файли контексту (ієрархічна пам'ять)

Файли контексту (за замовчуванням `GEMINI.md`) дозволяють надавати моделі
специфічні для проекту інструкції, гайди по стилю коду або будь-яку іншу фонову
інформацію.

- **Глобальний контекст:** `~/.gemini/GEMINI.md` — інструкції для всіх ваших
  проектів.
- **Контекст проекту:** `GEMINI.md` у корені проекту — правила для всієї команди
  або проекту.
- **Контекст підкаталогів:** `GEMINI.md` у папках модулів — дуже специфічні
  інструкції для окремих частин коду.

Ви можете переглянути об'єднаний контекст за допомогою команди `/memory show`
або оновити його за допомогою `/memory refresh`.

## Використання статистики

Для покращення Gemini CLI ми збираємо анонімну статистику використання (виклики
інструментів, запити до API, конфігурація сесії). Ми **не збираємо** персональні
дані, вміст підказок, відповідей або файлів.

Ви можете вимкнути збір статистики у `settings.json`:

```json
{
  "privacy": {
    "usageStatisticsEnabled": false
  }
}
```
