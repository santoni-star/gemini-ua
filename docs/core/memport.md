# Memory Import Processor (Процесор імпорту пам'яті)

Memory Import Processor — це функція, яка дозволяє модульно організовувати файли
`GEMINI.md`, імпортуючи вміст з інших файлів за допомогою синтаксису `@file.md`.

## Огляд

Ця можливість дозволяє розбивати великі файли `GEMINI.md` на менші, зручніші
компоненти, які можна використовувати повторно. Процесор підтримує як відносні,
так і абсолютні шляхи та має вбудовані засоби безпеки для запобігання циклічному
імпорту.

## Синтаксис

Використовуйте символ `@`, після якого вкажіть шлях до файлу:

```markdown
# Головний файл GEMINI.md

Ось основний зміст.

@./components/instructions.md

Більше контенту тут.

@./shared/configuration.md
```

## Безпека

- **Виявлення циклічного імпорту:** Процесор автоматично блокує ситуації, коли
  файл A імпортує файл B, а той знову імпортує файл A.
- **Безпека доступу:** Функція `validateImportPath` гарантує, що імпорт
  дозволений лише з певних каталогів, запобігаючи доступу до конфіденційних
  файлів системи.
- **Максимальна глибина:** Щоб уникнути нескінченної рекурсії, встановлено ліміт
  глибини імпорту (типово — 5 рівнів).

## Дерево імпорту

Процесор повертає ієрархічне дерево, яке показує, які файли були зчитані. Це
допомагає при налагодженні складних структур `GEMINI.md`.

Приклад структури:

```
Memory Files
 L project: GEMINI.md
            L a.md
              L b.md
            L included.md
```

## Найкращі практики

1. **Описові назви:** Давайте зрозумілі імена файлам-компонентам.
2. **Не зловживайте глибиною:** Намагайтеся тримати ланцюжки імпорту короткими.
3. **Відносні шляхи:** Використовуйте відносні шляхи (`./`) для кращої
   портативності проекту.

## Усунення несправностей

- **Імпорт не працює:** Перевірте, чи існує файл і чи правильний шлях.
- **Помилки прав доступу:** Переконайтеся, що файли доступні для читання та
  знаходяться у межах дозволених каталогів проекту.
