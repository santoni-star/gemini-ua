# Публікація розширень

Існує два основні способи розповсюдження розширень серед користувачів:

- [Git-репозиторій](#публікація-через-git-репозиторій)
- [Релізи GitHub](#публікація-через-релізи-github)

Релізи через Git є найпростішим та найгнучкішим підходом, тоді як релізи GitHub
можуть бути ефективнішими при першому встановленні, оскільки вони постачаються
як єдиний архів.

## Публікація через Git-репозиторій

Це найгнучкіший варіант. Вам достатньо створити публічний репозиторій, і
користувачі зможуть встановити ваше розширення за допомогою:
`gemini-ua extensions install <uri-репозиторію>`.

Вони можуть залежати від конкретної гілки або тегу через аргумент `--ref=<ref>`
(типово використовується гілка за замовчуванням). Щоразу, коли ви надсилаєте
коміти в цю гілку, користувачі отримають пропозицію оновити розширення.

## Публікація через релізи GitHub

Розширення можуть поширюватися через
[GitHub Releases](https://docs.github.com/en/repositories/releasing-projects-on-github/about-releases).
Це забезпечує швидше встановлення, оскільки не потребує клонування всього
репозиторію.

### Платформозалежні архіви

Якщо ваше розширення містить бінарні файли для конкретних ОС, ви повинні
дотримуватися такої схеми іменування архівів:
`{platform}.{arch}.{name}.{extension}`

- **platform**: `darwin` (macOS), `linux`, `win32` (Windows).
- **arch**: `x64`, `arm64`.
- **extension**: `.tar.gz` або `.zip`.

Приклад: `linux.x64.my-tool.tar.gz`. Файл `gemini-extension.json` обов'язково
має бути в корені кожного архіву.
