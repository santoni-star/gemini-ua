# Вибір моделі Gemini CLI (команда `/model`)

Виберіть модель для Gemini CLI. Команда `/model` дозволяє налаштувати модель,
яку використовує Gemini CLI, що дає вам більше контролю над результатами.
Використовуйте моделі **Pro** для складних завдань і міркувань, моделі **Flash**
для швидких результатів або (рекомендований) параметр **Auto**, щоб система сама
обирала найкращу модель для ваших завдань.

> **Примітка:** Команда `/model` (та прапорець `--model`) не перекривають
> модель, яку використовують субагенти. Отже, навіть при використанні прапорця
> `--model` ви можете бачити інші моделі у звітах про використання.

## Як використовувати команду `/model`

Введіть наступну команду в Gemini CLI:

```
/model
```

Запуск цієї команди відкриє діалогове вікно з наступними варіантами:

| Варіант           | Опис                                                                   | Моделі                                                        |
| ----------------- | ---------------------------------------------------------------------- | ------------------------------------------------------------- |
| Auto (Gemini 3)   | Дозвольте системі обрати найкращу модель Gemini 3 для вашого завдання. | gemini-3-pro-preview, gemini-3-flash-preview (якщо увімкнено) |
| Auto (Gemini 2.5) | Дозвольте системі обрати найкращу модель Gemini 2.5 для завдання.      | gemini-2.5-pro, gemini-2.5-flash                              |
| Manual (Ручний)   | Виберіть конкретну модель.                                             | Будь-яка доступна модель.                                     |

Ми рекомендуємо обирати один із варіантів **Auto**. Проте ви можете вибрати
**Manual**, щоб вказати конкретну модель зі списку доступних.

### Gemini 3 та попередні функції (Preview Features)

> **Примітка:** Gemini 3 наразі доступний не для всіх типів облікових записів.
> Щоб дізнатися більше, дивіться
> [Gemini 3 у Gemini CLI](../get-started/gemini-3.md).

Щоб увімкнути Gemini 3 Pro та Gemini 3 Flash (якщо доступно), увімкніть
[**Preview Features** за допомогою команди `/settings`](../cli/settings.md).

Ви також можете використовувати прапорець `--model`, щоб вказати певну модель
Gemini при запуску. Докладніше дивіться у
[документації з конфігурації](./configuration.md).

Зміни цих налаштувань будуть застосовані до всіх наступних взаємодій у сесії.

## Найкращі практики вибору моделі

- **За замовчуванням — Auto.** Для більшості користувачів режим _Auto_
  забезпечує баланс між швидкістю та продуктивністю, автоматично обираючи
  правильну модель залежно від складності завдання. Приклад: Розробка
  веб-додатка може містити як складні завдання (архітектура), так і прості
  (генерація CSS).

- **Перемикайтеся на Pro, якщо не отримуєте бажаних результатів.** Якщо ви
  вважаєте, що моделі потрібно бути "розумнішою", виберіть Pro вручну. Pro
  забезпечує найвищий рівень міркувань та креативності. Приклад: Складне
  багатоетапне налагодження коду.

- **Використовуйте Flash або Flash-Lite для швидких відповідей.** Якщо вам
  потрібна проста відповідь миттєво, Flash — найкращий вибір. Приклад:
  Конвертація об'єкта JSON у рядок YAML.
