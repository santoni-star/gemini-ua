# Огляд пакетів

Цей монорепозиторій містить два основні пакунки: `@google/gemini-cli` та
`@google/gemini-cli-core`.

## `@google/gemini-cli`

Це основний пакет Gemini CLI. Він відповідає за інтерфейс користувача, розбір
команд та всі інші функції, з якими взаємодіє користувач.

Коли цей пакет публікується, він об'єднується (bundled) в один виконуваний файл.
Цей бандл включає всі залежності пакета, зокрема й `@google/gemini-cli-core`. Це
означає, що незалежно від того, чи встановлює користувач пакет через
`npm install -g`, чи запускає через `npx`, він використовує один автономний
файл.

## `@google/gemini-cli-core`

Цей пакет містить основну логіку для взаємодії з Gemini API. Він відповідає за
виконання запитів до API, обробку аутентифікації та керування локальним кешем.

Цей пакет не об'єднується в один файл. Він публікується як стандартний пакет
Node.js зі своїми залежностями. Це дозволяє використовувати його як окрему
бібліотеку в інших проектах, якщо це необхідно.

## Робочі простори (NPM Workspaces)

Цей проект використовує
[NPM Workspaces](https://docs.npmjs.com/cli/v10/using-npm/workspaces) для
керування пакунками всередині монорепозиторію. Це спрощує розробку, дозволяючи
керувати залежностями та запускати сценарії для декількох пакунків з кореня
проекту.

### Переваги робочих просторів

- **Спрощене керування залежностями**: Запуск `npm install` у корені проекту
  встановлює залежності для всіх пакунків відразу.
- **Автоматичне пов'язування (Linking)**: Пакунки всередині робочого простору
  можуть залежати один від одного. Зміни в одному пакеті стають миттєво
  доступними іншим пакункам, які його використовують.
- **Зручний запуск сценаріїв**: Ви можете запускати скрипти будь-якого пакунка з
  кореня, використовуючи прапорець `--workspace`. Наприклад:
  `npm run build --workspace @google/gemini-cli`.
