# Конфігурація Gemini CLI (Застарілий формат v1)

**Примітка щодо застарілого формату конфігурації**

Цей документ описує застарілий формат v1 для файлу `settings.json`. Цей формат
більше не рекомендується до використання.

- Новий формат підтримується у стабільних версіях починаючи з **[09/10/25]**.
- Автоматична міграція зі старого формату на новий розпочнеться **[09/17/25]**.

Для отримання детальної інформації про новий рекомендований формат, будь ласка,
дивіться [поточну документацію з налаштування](./configuration.md).

Gemini CLI пропонує кілька способів налаштування своєї поведінки, включаючи
змінні середовища, аргументи командного рядка та файли налаштувань. У цьому
документі описано різні методи конфігурації та доступні параметри.

## Рівні конфігурації

Налаштування застосовуються в такому порядку пріоритетності (нижчі номери
перекриваються вищими):

1.  **Значення за замовчуванням:** Жорстко закодовані значення всередині
    програми.
2.  **Файл системних значень за замовчуванням:** Загальносистемні налаштування,
    які можуть бути змінені іншими файлами.
3.  **Файл налаштувань користувача:** Глобальні налаштування для поточного
    користувача.
4.  **Файл налаштувань проекту:** Специфічні налаштування для конкретного
    проекту.
5.  **Файл системних налаштувань:** Загальносистемні налаштування, що мають
    найвищий пріоритет.
6.  **Змінні середовища:** Системні змінні або змінні сеансу, які також можуть
    завантажуватися з файлів `.env`.
7.  **Аргументи командного рядка:** Значення, що передаються безпосередньо при
    запуску CLI.

## Файли налаштувань

Gemini CLI використовує файли JSON для збереження конфігурації. Ці файли можуть
знаходитися у чотирьох місцях:

- **Файл системних значень за замовчуванням:**
  - **Розташування:** `/etc/gemini-cli/system-defaults.json` (Linux),
    `C:\ProgramData\gemini-cli\system-defaults.json` (Windows) або
    `/Library/Application Support/GeminiCli/system-defaults.json` (macOS).
  - **Область дії:** Базовий шар налаштувань для всієї системи.
- **Файл налаштувань користувача:**
  - **Розташування:** `~/.gemini/settings.json`.
  - **Область дії:** Застосовується до всіх сесій Gemini CLI для поточного
    користувача.
- **Файл налаштувань проекту:**
  - **Розташування:** `.gemini/settings.json` у кореневому каталозі вашого
    проекту.
  - **Область дії:** Застосовується лише при запуску Gemini CLI з цього проекту.
- **Файл системних налаштувань:**
  - **Розташування:** `/etc/gemini-cli/settings.json`.
  - **Область дії:** Застосовується до всіх користувачів системи. Має вищий
    пріоритет над усіма іншими файлами налаштувань.

**Примітка щодо змінних середовища:** Рядкові значення у файлах `settings.json`
можуть посилатися на змінні середовища за допомогою синтаксису `$VAR_NAME` або
`${VAR_NAME}`.

> **Примітка для корпоративних користувачів:** Для отримання вказівок щодо
> розгортання та керування Gemini CLI у корпоративному середовищі, будь ласка,
> дивіться документацію [Корпоративна конфігурація](../cli/enterprise.md).

### Доступні налаштування (v1):

- **`contextFileName`** (рядок або масив рядків): Назва файлів контексту
  (типово: `GEMINI.md`).
- **`fileFiltering`** (об'єкт): Керує врахуванням `.gitignore` та рекурсивним
  пошуком.
- **`coreTools`** (масив рядків): Список основних інструментів, доступних
  моделі.
- **`allowedTools`** (масив рядків): Список інструментів, що виконуються без
  підтвердження.
- **`theme`** (рядок): Візуальна тема.
- **`vimMode`** (boolean): Увімкнення режиму vim.
- **`sandbox`** (boolean або рядок): Налаштування пісочниці.
- **`mcpServers`** (об'єкт): Налаштування серверів MCP.

### Усунення несправностей продуктивності пошуку файлів

Якщо ви відчуваєте затримки при пошуку файлів (наприклад, автодоповнення `@`):

1.  **Використовуйте `.geminiignore`:** Виключіть папки з великою кількістю
    файлів (`node_modules`, `dist`).
2.  **Вимкніть нечіткий пошук:** Встановіть `disableFuzzySearch: true`.
3.  **Вимкніть рекурсивний пошук:** Встановіть
    `enableRecursiveFileSearch: false`.
