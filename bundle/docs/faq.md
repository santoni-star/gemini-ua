# Часті запитання (FAQ)

Ця сторінка містить відповіді на поширені запитання та рішення проблем, що
виникають під час використання Gemini CLI.

## Загальні питання

### Чому я отримую помилку `API error: 429 - Resource exhausted`?

Ця помилка означає, що ви перевищили ліміт запитів до API. Gemini API має
обмеження швидкості для запобігання зловживанням та забезпечення справедливого
використання.

Щоб вирішити це:

- **Перевірте використання:** Перегляньте статистику у Google AI Studio або
  панелі керування Google Cloud.
- **Оптимізуйте підказки:** Якщо ви робите багато запитів за короткий час,
  спробуйте об'єднувати їх або додавати затримки.
- **Запитуйте збільшення квоти:** Якщо вам постійно потрібен вищий ліміт, ви
  можете подати запит на збільшення квоти у Google.

### Чому я не бачу кількість кешованих токенів у статистиці?

Інформація про кешовані токени відображається лише тоді, коли вони
використовуються. Ця функція доступна для користувачів ключів API (Gemini API
key або Google Cloud Vertex AI), але недоступна для користувачів OAuth (особисті
або корпоративні акаунти Google). Це пов'язано з тим, що Code Assist API наразі
не підтримує кешування вмісту.

## Встановлення та оновлення

### Як оновити Gemini CLI до останньої версії?

Якщо ви встановлювали глобально через `npm`, виконайте:

```bash
npm install -g santoni-star/gemini-ua
```

Якщо ви компілювали з вихідного коду, стягніть останні зміни з репозиторію
(`git pull`) та перезберіть проект (`npm run build`).

## Питання конфігурації

### Як налаштувати `GOOGLE_CLOUD_PROJECT`?

Ви можете встановити ID проекту через змінну середовища у вашій оболонці:

```bash
export GOOGLE_CLOUD_PROJECT="ваш-id-проекту"
```

Щоб зробити це налаштування постійним, додайте цей рядок у файл `~/.bashrc` або
`~/.zshrc`.

### Який найкращий спосіб безпечного зберігання ключів API?

Відкрите вказання ключів у скриптах або їх збереження в Git є ризиком для
безпеки.

- **Використовуйте файл `.env`:** Створіть файл `.env` у каталозі `.gemini`
  вашого проекту (`.gemini/.env`) і запишіть ключі туди. Gemini CLI автоматично
  завантажить ці змінні.
- **Використовуйте системне сховище ключів:** Для максимальної безпеки
  використовуйте інструменти керування секретами вашої ОС (Keychain у macOS,
  Credential Manager у Windows).

### Де зберігаються файли налаштувань Gemini CLI?

Налаштування зберігаються у двох файлах `settings.json`:

1. У домашньому каталозі: `~/.gemini/settings.json`.
2. У корені проекту: `./.gemini/settings.json`.

## Google AI Pro/Ultra та підписки

### Чи буде Gemini використовувати мої дані для навчання моделей, якщо я куплю платну підписку?

Google не використовує ваші дані для покращення своїх моделей машинного
навчання, якщо ви користуєтеся платним планом. Навіть у безкоштовній версії ви
можете відмовитися від використання ваших даних для навчання у налаштуваннях
конфіденційності.
